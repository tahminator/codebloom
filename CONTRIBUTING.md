# **Pull Request Guidelines**

## Before You Open a PR

> [!NOTE]
> If you aren’t sure whether or not you’re doing the right thing, please create the PR anyways. We’re happy to accept any contribution, and we’ll help you make any necessary changes that may be needed!

# **Review Process**

- The PR checklist is for developers. If you open a PR, we’ll work with you to complete it.

- All pull requests will be reviewed by a developer from the Codebloom team.

- Please try to refrain from @ing developers. We aim to respond as quickly as we can, but larger or more complex PRs may take a bit longer.
